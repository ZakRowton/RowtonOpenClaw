# memory.md - What to Remember

_Keep this small. It’s injected at the start of every session._

Use it for:

- **Facts** — Names, preferences, decisions you and your user have agreed on
- **Preferences** — How they like replies (short vs long, tone, format)
- **Ongoing context** — Current projects, constraints, or rules that should apply every time

Avoid dumping long logs or one-off notes here. Prefer `memory/YYYY-MM-DD.md` for daily notes; keep `memory.md` as a short, stable “core” that rarely changes.

---

Save this file at the workspace root as `memory.md` (or `MEMORY.md`). Either name is recognized.
